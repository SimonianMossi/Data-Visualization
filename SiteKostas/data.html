Millions of lines of code infographic

Below is an infographic from information is beautiful, concept & design: David McCandless, research: Pearl Doughty-White, Miriam Quick.

It is a barchart-type comparison of codebase sizes of various software. Different entries are colour-coded into groups (apps, OS, browsers, games, machines). Different versions of the same software are marked by "leaping" rings.

I like this infographic because:

  • It is a rough representation of how much work goes into certain kinds of software that we may have not thought of.
  • It is intuitive and easy to understand.
  • It is "clean", uncluttered and aesthetically pleasing. Whether we like it or not, people will be less inclined to study an ugly/messy presentation, even if the data behind it is interesting.
  • Most of all, perhaps inadvertently, it makes a point to show that bigger code is not always better code. Windows Vista has more code than its successor, Windows 7, but has been criticised as significantly inferior. The Space Shuttle boasts a mere 40k lines of code. At the end of the day, file size and lines of code are just a number, and your code is judged by what it does, and how well it does it.
